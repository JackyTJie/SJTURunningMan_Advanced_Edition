## 简介

本项目仅用于学习、研究和技术测试，主要用于理解运动数据记录与上传相关的技术流程。项目作者不鼓励、不支持任何违反学校规定的使用方式。请在使用前确认自己的行为符合 SJTU 及相关平台的要求，请不要用于伪造运动数据或规避体育锻炼要求。

Windows 版提供跑步次数、时间、距离和路线等参数配置，便于进行流程研究、规则测试和结果复核。程序默认会以昨日作为结束日期，向前生成指定次数的跑步记录（默认为1次），每次跑步4km，时间为08:00:00。

> p.s. 软件的地图自定义路线功能，基于从百度地图获取的免费API开发，现在的API使用时间略长（超过半年）可能会受到限制或者封禁，届时自定义路线功能可能无法使用。

## 功能特性

### 1. 电脑版图形用户界面

**用户配置区域:**

- **用户名输入框**: 输入您的Jaccount用户名
- **密码输入框**: 输入您的Jaccount密码（密码输入为星号隐藏显示）

**上传设置区域:**

- **跑步次数**: 直接输入正整数，范围为1-30条记录
- **跑步时间**: 直接输入时间，格式为HH:MM:SS（默认08:00:00）
  - 每次跑步时间会自动添加随机波动（±10分钟和±30秒），使跑步时间更真实自然
- **结束日期**: 直接输入日期，格式为YYYY-MM-DD（默认为昨天），系统会以这天作为最新一条记录并向前生成
- **跑步距离**: 直接输入距离，单位为km，最大不超过4km（默认4km）
- **预设路线**: 选择内置 `default.txt`、旧版 `user.txt` 或通过“自定义...”导入外部 txt 路线；鼠标滚轮不会误切换路线

**功能按钮:**

- **开始生成并上传按钮**: 执行完整的跑步数据生成和上传流程
- **停止任务按钮**: 在上传过程中停止当前任务
- **设计/更新路线按钮**: 打开浏览器路线规划器进行自定义路线绘制，生成 txt 后可在“预设路线”中导入
- **查看本次路线按钮**: 在系统浏览器中打开本条实际上传路线预览
- **GitHub按钮**: 打开项目 GitHub 仓库, 欢迎大家来contribute
- **水源按钮**: 打开水源社区相关讨论帖
- **说明按钮**: 显示开发者信息

**实时反馈区域:**

- **进度条**: 显示上传进度（已完成/总任务数）
- **状态标签**: 显示当前操作状态
- **启动警告/日志区域**: 未运行任务时显示使用警告；任务开始后切换为详细操作日志
- **路线预览摘要**: payload 生成后显示距离、点数、跳段数量和上传状态

### 2. 轨迹风险指数与实际路线预览

**生成后自检:**

- 每条跑步 payload 生成完成后、上传前会自动运行轨迹风险指数检测
- 检测结果包含 `0-100` 分风险分、低/中/高风险等级、轨迹统计和主要原因
- 低风险和中风险会写入日志；高风险会在上传前弹窗确认，用户可继续或取消本条上传
- 结构性异常（例如有效点不足、时间戳非递增、严重跳点）会被显著标记

**可解释检测维度:**

- 时间采样规律性
- 速度过度稳定
- 线性插值痕迹
- GPS 噪声/精度异常
- 路线重复与机械往返
- 停顿/状态一致性

**实际上传路线预览:**

- 预览使用已生成并准备上传的 payload 坐标，而不是原始 txt 路线
- 地图中展示主路线、起点、终点、距离、点数、时长、最长单段和大跳段统计
- 大于 150m 的跳段会以红色高亮，便于发现路线文件或生成结果异常
- 预览 HTML 通过系统浏览器打开，避免额外打包 Qt WebEngine 导致 Windows exe 体积明显增大

### 3. 路线规划器

**图形化路线自定义:**

- 点击"设计/更新路线"按钮打开路线规划器
- 在百度地图界面中点击选择路径坐标点，形成跑步路线
- 支持添加、删除、清空坐标点
- 点击"下载路线 txt"按钮将下载 `custom_route.txt` 文件
- 回到 Windows 主界面，在"预设路线"下拉菜单中选择"自定义..."并导入该 txt 文件
- 导入后的路线会复制到软件所在目录的 `Routes/` 文件夹，重启软件后仍可选择
- 如 exe 同目录存在旧版 `user.txt`，仍会作为 `user.txt（自定义路线）` 显示以保持兼容

**坐标点管理:**

- 点击地图任一点自动添加坐标点并显示在地图上
- 点击地图上的标记点可删除该坐标点
- 左侧面板显示所有选中的坐标点列表
- 支持清空所有点的功能

**内置路线说明:**

- 当前 Windows 版内置路线暂仅保留 `assets/Routes/default.txt`
- 其他历史内置路线已移动到 `assets/Routes/NeedsReview/`，待人工校验跳段、长度和起终点问题后再恢复
- 用户自行导入的路线不会被隔离，仍会复制到 exe 同目录 `Routes/` 文件夹

### 4. 数据生成与上传逻辑

**智能路径调整:**

- 根据目标距离和速度自动调整路线长度
- 如果路线长度超过目标距离，提供选择是否自动削减路线
- 如果路线较短，采用往返或环路策略以达到目标距离
- 起终点距离大于15米时使用往返策略，否则使用环路策略

**GPS坐标校正:**

- 自动应用GPS坐标偏移校正以匹配实际地图数据
- 包含额外微调校正以解决地图选择坐标与实际记录的偏差

**轨迹分段处理:**

- 将整体轨迹拆分为多个轨迹段，模拟真实的跑步过程
- 为各段分配不同的状态（正常、停止、无效）
- 包含合理的轨迹点时间戳生成

### 5. 上传流程

**三步上传流程:**

1. **获取认证信息**: 获取认证令牌和跑步规则
2. **生成跑步数据**: 根据配置生成符合要求的跑步轨迹数据
3. **上传跑步数据**: 将生成的数据上传到服务器

**多天上传支持:**

- 支持一次性上传多天的跑步数据
- 数据生成为过去N天（从昨天开始的连续N天）
- 每天对应一个独立的上传请求

**上传控制:**

- 显示每条数据的上传进度
- 记录成功和失败的上传次数
- 支持中途停止上传过程
- 每条上传前会更新路线预览摘要和风险自检结果
- 高风险轨迹默认不会静默上传，会先弹窗确认

### 6. 安全与验证

**验证码处理:**

- 自动处理实时验证码识别（通过外部模型）
- 无需用户手动输入验证码

**凭据验证:**

- 支持通过`--verify-credentials`验证凭据有效性而不上传数据
- 提供登录失败的详细错误信息

**错误处理:**

- 全面的异常处理机制
- 详细的错误日志输出
- 支持中断操作的安全处理

**本地透明性:**

- 轨迹风险分析和路线预览均在本地运行，不会修改 payload 本身
- 预览 HTML 写入系统临时目录后由浏览器打开，不依赖内嵌 WebEngine

## 手机版使用指南

### 1. 安装安卓应用

1. 下载 APK 文件
2. 将 APK 文件传输到您的 Android 设备
3. 在设备上安装 APK（可能需要开启"允许安装未知来源应用"选项）

### 2. 登录流程

1. 应用会自动打开jaccount登录页面获取cookie, 您可以手动登陆或交我办一键登陆
2. 应用会自动检测cookie有效性，若失效自动重新登录，若仍有问题请在设置清除应用数据


### 3. 配置跑步参数

- **上传天数**：选择需要上传的跑步天数（1-25天）
- **跑步时间**：选择跑步发生的时间（6:00-23:00）
- **开始日期**：设置跑步开始的日期（默认为昨天）
- **跑步距离**：选择每次跑步的距离（1-5公里）
- **自定义路线**：默认为思源湖浪漫跑步路线，可以在新的页面连点设置，点越多越好，**注意：点是有顺序的，且起始与终止点越接近越好**

### 4. 开始上传

- 点击"一键跑步"按钮开始上传
- 观察进度条和状态提示了解上传进度
- 上传完成后会显示成功/失败统计

## 常见问题与故障排除

### 上传失败处理
此版本已兼容异地登陆
如果遇到上传失败的情况，请按以下步骤尝试解决：

1. **清除应用数据**：
   - 进入手机 **设置** → **应用管理**
   - 找到本应用（SJTU 一键校园跑）
   - 点击 **存储** → **清除数据** 和 **清除缓存**
   - 重新打开应用并重新输入 Cookie

2. **检查 Cookie 有效性**：
   - Cookie 可能已过期（通常有效期为几小时到几天）
   - 重新登录交我办获取新的 Cookie
   - 确保复制的 Cookie 完整无遗漏

3. **网络连接检查**：
   - 确保设备连接到稳定的网络
   - 尝试切换 WiFi/移动数据
   - 检查是否被校园网限制

4. **服务器状态**：
   - 可能是交我办服务器维护或故障
   - 等待一段时间后重试

### 其他注意事项

- **跑步次数**：第一次尝试时，跑步次数建议选择1天来测试有效性安全性
- **Cookie 有效期**：Cookie 通常有有效期限制，过期后需要重新获取
- **多设备登录**：同一账号在多个设备同时登录可能导致 Cookie 失效
- **安全提醒**：Cookie 包含您的登录凭证，请勿分享给他人

## 源代码使用方法


1. **启动GUI界面**:

   ```bash
   python qtui.py
   ```

2. **填写用户信息**:
   - 在"用户名"输入框中输入您的Jaccount学号
   - 在"密码"输入框中输入您的Jaccount密码

3. **设置上传参数**:

   - 在"跑步次数"中输入记录数量（1-30条）
   - 在"跑步时间"中输入时间，格式为 HH:MM:SS，例如 08:00:00
   - 在"结束日期"中输入日期，格式为 YYYY-MM-DD，例如 2026-05-07
   - 在"跑步距离"中输入距离，单位为 km，最多 4km
   - 在"预设路线"中选择内置 `default` 路线，或选择"自定义..."导入 txt 路线文件

4. **开始上传**:
   - 点击"开始生成并上传"按钮开始上传
   - 观察进度条、日志输出、风险检测摘要和路线预览摘要了解上传状态
   - 如需查看本条实际上传轨迹，点击"查看本次路线"
   - 如需停止，点击"停止任务"按钮

5. **自定义路线**:
   - 点击"设计/更新路线"按钮打开路线规划器
   - 在地图上点击选择坐标点
   - 点击"下载路线 txt"下载 `custom_route.txt` 文件
   - 回到主界面，在"预设路线"中选择"自定义..."导入该 txt 文件


### Windows 打包方法

请使用仓库提供的 `main.spec` 或 `build_windows.ps1` 打包，避免直接执行 `pyinstaller qtui.py` 生成默认图标。

```powershell
powershell -ExecutionPolicy Bypass -File .\build_windows.ps1
```

打包脚本会从仓库根目录调用：

```powershell
python -m PyInstaller --clean --noconfirm .\main.spec
```

`main.spec` 会使用 `assets/SJTURM.ico` 作为 exe 图标，并把 `assets/` 目录一起打入程序。


### 路线自定义方法

- 点击"设计/更新路线"按钮
- 在打开的浏览器中点击地图选择路线坐标点
- 点击"下载路线 txt"下载 `custom_route.txt` 文件
- 回到 Windows 主界面，在"预设路线"中选择"自定义..."导入该 txt 文件
- 导入后的路线会复制到软件所在目录的 `Routes/` 文件夹；如目标文件已存在，会自动追加 `_1` 等序号避免覆盖


## 重要提醒

为获得最佳效果，建议：

- 尽量避免选择1公里距离的跑步(选1km大概率最后不满1km)，推荐选择2公里或以上的距离，以获得更稳定的结果
- Windows 图形界面中每次跑步距离最多为4km；请合理设置目标距离，避免触发系统验证机制
- Windows 图形界面启动时需要阅读并勾选承诺书；请确认用途符合校规和相关法律法规
- 内置路线暂仅保留 `default.txt`；`assets/Routes/NeedsReview/` 中的路线待校验后再恢复
- 每次上传前建议查看“路线预览”和风险自检结果，确认路线距离、跳段和轨迹状态符合预期
- 如需精确控制跑步距离，请多次尝试以找到最适合的设置
- 自定义路线时，确保路线长度与目标距离匹配，避免过长或过短；路线 txt 格式应为每行 `longitude,latitude`
